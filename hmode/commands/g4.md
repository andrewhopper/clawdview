# G4 Backlog Processor

Process launched partner opportunities from 2025 and backfill Partner Solution Engagement activities for G4 goal tracking.

## Goal: G4 - Partner Solutions Adoption

Track AWS Originated launched utility opportunities with:
- A 'Partner Solution Engagement [Architecture]' SA/CSM activity
- A partner solution opportunity tag

## Workflow

1. **Get Account**
   - Ask user for account name or ID
   - Search and confirm account

2. **Find Partner Opportunities**
   - Search for opportunities on the account
   - Filter criteria:
     - Created date: 2025-01-01 to 2025-12-31
     - Stage: "Launched" or probability = 100
     - Partner attached (check description, name, or tags for partner mentions)
   - Present list of matching opportunities

3. **Select Opportunities**
   - Show each opportunity with:
     - Name
     - Amount
     - Created date
     - Close date
     - Partner name (detected)
   - Ask user to select which opportunities need G4 activities

4. **For Each Selected Opportunity**
   - Check if "Partner Solution Engagement [Architecture]" activity already exists
   - If exists: skip and note
   - If not exists: offer to create

5. **Create Tech Activity**
   - **saActivity**: "Partner Solution Engagement [Architecture]"
   - **subject**: "Recommend {Partner} for {Customer}"
   - **activityDate**: Created date + 7 days (within 2 weeks of opp start)
   - **status**: "Completed"
   - **isVirtual**: true
   - **parentRecord**: opportunity ID
   - **description**: Include:
     - Partner name
     - Opportunity details
     - Solution description (from opp description)
     - "G4 backfill activity - Partner solution engagement"
   - **services**: Extract from opportunity name/description
   - **domains**: Suggest based on opportunity context

6. **Verify G4 Requirements**
   - Check opportunity has valid estimated value (not $0, $1, or placeholder)
   - Check opportunity products are listed
   - Warn if missing and offer to skip or proceed

7. **Summary Report**
   - Show count of activities created
   - Show count of opportunities skipped (already logged)
   - Show count of opportunities with issues (missing value/products)

## Partner Detection Strategy

Extract partner names from:
1. Opportunity name (e.g., "CloudZero - Solution [Caylent]")
2. Opportunity description (mentions of partner companies)
3. Lead source = "Partner Referral"
4. Opportunity tags (if available)

Common partner indicators:
- Partner name in brackets: [Caylent], [Loka]
- Partner referral in description
- Co-sell opportunity mentions

## Date Calculation

Activity date = Opportunity Created Date + 7 days
- Ensures activity falls within 2 weeks of opportunity start
- Represents engagement that led to the opportunity

## Validation Checks

Before creating activity, verify:
- ✅ Opportunity stage is "Launched" or "Completed"
- ✅ Opportunity has valid amount (not $0, $1)
- ✅ Partner is clearly identified
- ✅ No existing "Partner Solution Engagement" activity
- ⚠️ Warn if products not listed (but allow proceed)

## Implementation Notes

- Use aws-sentral MCP tools:
  - `search_accounts` - find customer account
  - `get_opportunities_for_account` - retrieve opportunities (includeClosed=true)
  - `list_opportunity_tasks` - check for existing activities
  - `create_tech_activity` - create Partner Solution Engagement activity
  - `get_opportunity_line_items` - verify products listed

- Filter opportunities in code:
  - Parse createdDate: must be in 2025
  - Check stageName contains "Launched" OR probability = 100
  - Detect partner from name/description/leadSource

- Activity date calculation:
  ```python
  from datetime import datetime, timedelta
  created_date = datetime.fromisoformat(opp['createdDate'])
  activity_date = created_date + timedelta(days=7)
  activity_date_str = activity_date.strftime('%Y-%m-%d')
  ```

- Always set:
  - `saActivity`: "Partner Solution Engagement [Architecture]"
  - `isVirtual`: true
  - `status`: "Completed"

- Batch processing:
  - Process opportunities one at a time
  - Show progress: [N/Total]
  - Allow user to skip individual opportunities

## G4 Requirements Checklist

For each opportunity, verify:
- [ ] Partner Solution Engagement [Architecture] activity recorded
- [ ] Partner solution opportunity tag applied (if API available)
- [ ] Valid opportunity estimated value (not $0, $1, or placeholder)
- [ ] Opportunity products listed and accurate (warn if missing)
- [ ] AWS Originated launched utility opportunity (confirmed by stage)

## Error Handling

- If account not found: ask for different search term
- If no opportunities found: inform user and exit
- If partner detection fails: ask user to manually specify partner
- If activity creation fails: show error, ask to skip or retry
- If validation fails: warn but allow user to proceed or skip
