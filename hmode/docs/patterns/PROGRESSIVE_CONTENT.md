# Progressive Content Creation

**Purpose:** Stage-gated content creation for complex artifacts

**When to use:**
- Detailed presentations (>10 slides, complex content)
- Comprehensive written documents (>5 pages, high rigor)
- Complex flowcharts/diagrams (>10 nodes, multi-layer)
- Strategic documents (PRDs, 6-pagers, whitepapers)
- Technical specifications (detailed design docs)

**Workflow:**
Use `/progressive-content` slash command for 6-stage workflow with approval gates.

**See:** `hmode/commands/progressive-content.md` for implementation
