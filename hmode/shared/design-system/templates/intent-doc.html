Meeting Buddy

Planning
▼
Purpose

Create shared technical understanding between customers and sales teams through real-time visual architecture diagrams

▼
Key Tasks
KT-1

Technical entities are detected accurately from conversation audio

Expected Result: >90% accuracy on AWS service names (e.g., "ECS", "Load Balancer", "RDS") from audio transcription

KT-2

Users can confirm or correct low-confidence entity detections

Expected Result: UI presents ambiguous entities for confirmation; user can accept/reject/modify within 2 seconds

KT-3

Diagram history is preserved with ability to rewind to previous states

Expected Result: Event-sourced architecture allows stepping backward through conversation timeline; diagram updates reflect historical state

KT-4

System predicts likely architecture connections (e.g., Load Balancer → ECS more probable than Load Balancer → RDS)

Expected Result: Edge suggestions based on AWS best practices; system proposes next likely component with confidence score

KT-5

Architecture diagrams render in real-time as conversation progresses

Expected Result: Diagram updates within 1-2 seconds of entity detection; smooth animation for new nodes/edges

KT-6

Diagrams can be securely shared with meeting participants

Expected Result: Shareable link with access controls; participants see synchronized diagram view during meeting

KT-7

Users can manually edit diagram data and structure

Expected Result: Click-to-edit nodes/edges; changes persist and sync across all viewers in real-time

▼
End State
ES-1

Can conduct a customer meeting with real-time architecture visualization

ES-2

Both salesperson and customer see the same diagram simultaneously during the meeting

ES-3

Diagram accurately reflects the technical architecture discussed

ES-4

Meeting participants can export/share the final diagram after the meeting

ES-5

Audio transcription captures technical terms accurately (>90% accuracy on AWS service names)

ES-6

Zero audio data leaves secure boundary (customer privacy maintained)

ES-7

Diagram rewind function works correctly (can step back through conversation history)
