Rule Engine Architecture

Decoupled, pluggable system for deterministic AI validation and enforcement

1. Execution Flow

1
User Action
User requests AI to write file, generate response, or perform action
Write file: src/main.py
Response: "Would you like to deploy?"
↓
2
Claude Code Hook
frontgate_v2.py intercepts action
  • PreToolUse: Before tool execution (can block)
  • PostToolUse: After tool execution (warn only)
↓
3
Rule Engine
Engine filters and runs applicable rules based on context
engine = RuleEngine()
engine.register_rule(PhaseGateRule())
engine.register_rule(NumberedOptionsRule())

result = engine.validate(input_data)
↓
4
Rule Validation
Each applicable rule validates input independently
  • PhaseGateRule → Check if phase ≥ 8 for code files
  • NumberedOptionsRule → Check for [1], [2], [3] format
  • OneQuestionRule → Check for multiple questions
↓
5
Result Aggregation
Engine aggregates violations from all rules
ValidationResult(
is_valid=False,
violations=[RuleViolation(...)],
metadata={...}
)
↓
6
Action
Based on severity:
  • BLOCKER → Exit 1 (blocks execution)
  • WARNING → Print message, continue (exit 0)
  • INFO → Log only (for evals)

2. Built-in Rules

PhaseGateRule
⛔ Blocker
Blocks code file writes before Phase 8 (Implementation). Enforces SDLC phase transitions.
NumberedOptionsRule
⚠️ Warning
Enforces [1], [2], [3] format for multiple choice options. CLAUDE.md Section 3.4.
OneQuestionRule
⚠️ Warning
Detects multiple questions in single response. CLAUDE.md Section 3.2.
NoApologiesRule
ℹ️ Info
Detects unnecessary apologies ("I'm sorry", "I apologize"). For eval/quality scoring.
CitationCountRule
ℹ️ Info
Counts citations in research responses. Evaluates research quality.
ResponseLengthRule
ℹ️ Info
Checks response length (20-500 words). Evaluates conciseness and completeness.

3. Core Components

Rule (Abstract Base Class)
  • name → Unique identifier
  • description → Human-readable description
  • severity → BLOCKER | WARNING | INFO
  • applicable_contexts → List[ValidationContext]
  • validate(input) → Optional[RuleViolation]
RuleEngine
  • register_rule(rule) → Add rule to engine
  • unregister_rule(name) → Remove rule by name
  • get_rules(context) → List applicable rules
  • validate(input) → Run validation, return result
ValidationContext
  • RESPONSE → AI response text
  • PRE_TOOL → Before tool execution
  • POST_TOOL → After tool execution
  • FILE_WRITE → File write operation
  • PHASE_TRANSITION → SDLC phase change
