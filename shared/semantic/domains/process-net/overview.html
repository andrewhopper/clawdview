Process-Net Domain

Deterministic Agent Orchestration with Petri Net Semantics

v0.1.0 Schema Complete SDLC Validated Event Sourcing

The Problem

!
AI Agents Are Non-Deterministic
  • Same input can yield different execution paths
  • Debugging failures is difficult
  • No ability to "rewind" and try alternatives
  • Hard to reproduce issues
✓
The Solution
  • Force agents to follow deterministic graphs
  • Event sourcing for full replay/rewind
  • Priority queue of valid next actions
  • Formal Petri net semantics

The Priority Queue

GetEnabledTransitions returns all transitions that CAN fire, ordered by priority. The agent picks from the top. Fully deterministic.

#2 score: 70 t_request_approval — Request human approval ENABLED
#3 score: 50 t_skip_to_deploy — Skip to deployment BLOCKED

Event Sourcing

Every Fire action produces a Firing record. Full replay/rewind capability. 

// events.jsonl (append-only log)

{"id":"f-001","transition":"t_start","seq":1,"marking_after":{"p_seed_done":1},"actor":"claude-code","ts":"10:00:00"}
{"id":"f-002","transition":"t_research","seq":2,"marking_after":{"p_research_done":1},"actor":"claude-code","ts":"10:05:00"}
{"id":"f-003","transition":"t_skip_feasibility","seq":3,"marking_after":{"p_feasibility_done":1},"actor":"claude-code","ts":"10:05:01"}
{"id":"f-004","transition":"t_expansion","seq":4,"marking_after":{"p_expansion_done":1},"actor":"claude-code","ts":"10:10:00"}
...

// Replay to sequence 3:
ReplayTo(case_id, target_sequence=3)
→ Reconstructs marking: { p_feasibility_done: 1 }

// Fork from sequence 3:
ForkCase(case_id, fork_at=3, name="try-different-approach")
→ New case starting at p_feasibility_done
