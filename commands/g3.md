# G3 Security and Resilience Tracker

Track qualified Security and Resilience engagements for G3 goal tracking.

## Goal: G3 - Strengthen Security and Resilience

Track tech team activities performed in 2025 that are tagged as one of the qualified Security and Resilience engagements.

## Qualified G3 Engagements (2025)

All goals start with "G3 2025 - " prefix:

### Security Programs
- AWS Partner engagement focused on Security or Resiliency - SSR
- AWS Professional Services engagement focused on Security or Resiliency - SSR
- AWS Well-Architected Framework Review (WAFR) - SSR
- Landing Zone Assessment Accelerator - SSR
- Security & Compliance Visibility Review (SCVR) - SSR
- Security Health Improvement Program (SHIP) - SSR
- Security Improvement Program (SIP) - SSR
- Security Journeys - SSR
- Security Orientation Roundtable (SOR) - SSR
- TorchLight - SSR

### Resilience Programs
- Business Impact Analysis (BIA) - SSR
- Customer Chaos GameDay - SSR
- Customer Reliability Profiles - SSR
- Customer School of Resilience (C-SoR) - SSR
- Cyber Resilience Lab (CRL) - SSR
- Driving Resilience: Planning - Execution - Testing (DrPET) - SSR
- ReSCO - SSR
- Resilience Core Program (RCP) - SSR
- Resilience Maturity Assessment (RMA) - SSR
- Resilience Readiness Review (RRR) - SSR
- Simulated Conditions Response and Management (SCRaM) - SSR
- Threat Modelling - SSR

## ReSCO-Specific Guidance

**When to Log ReSCO Activities:**
- ONLY after presenting assessment results for Resilience, Security, or Cost Optimization pillars
- Do NOT log customer outreach or introduction presentations

**How to Log ReSCO Activities:**

1. **Create SA Activity against the Account:**
   - **Activity Type:**
     - "Well Architected [Architecture]" - for Security or Resilience findings
     - "Cost Optimization [Management]" - for Cost Optimization findings
   - **Subject Line:** MUST include "FY-25-ReSCO"
   - **Description:** Include:
     - Pillar name (Security, Resilience, or Cost Optimization)
     - Number of accounts scanned
     - Number of critical/medium/low findings
     - Recommendations (e.g., "enable GuardDuty", "turn on Multi-AZ RDS")

2. **For Security & Resilience ONLY:**
   - Add Goal tag: "G3 2025 - ReSCO - SSR"
   - Log separate activities for each pillar (one for Security, one for Resilience)

3. **Cost Optimization:**
   - Use "FY-25-ReSCO" in subject
   - Do NOT add the G3 SSR tag (not eligible for G3 goal tracking)

**ReSCO Opportunities:**

**When to Create:**
- ONLY if there's potential for AWS Native Services, ISV Solutions, or ProServe/Partner SoW adoption
- For Reserved Instance/Savings Plan purchases from Cost Optimization assessments

**How to Log:**
- Record Type: Utility
- Primary Campaign Source: FY-25-ReSCO
- Total Opportunity Value: Use AWS Calculator to estimate service usage

**Key Points:**
- One ReSCO Security + one ReSCO Resilience = 2 separate activities, both tagged with G3 SSR
- Cost Optimization assessments don't count toward SSR goals
- Don't create opportunities just for assessment introductions

## Workflow

1. **Get Context**
   - Ask: "Account or Opportunity?"
   - If Account: search and select account
   - If Opportunity: search and select opportunity

2. **Engagement Details**
   - Ask for engagement type (show numbered list of qualified G3 engagements)
   - Ask for completion date (default: today)
   - **Special handling for ReSCO:** If user mentions ReSCO or FY-25-ReSCO, confirm this is a ReSCO activity
   - Ask for description with prompts:
     - Impact of the engagement
     - Next steps
     - Customer feedback or quotes
     - ⚠️ Do NOT include sensitive security/resilience findings
   - **Allow "skip" option:** User can type "skip" to use a basic auto-generated description

3. **Activity Type Selection**
   - Show suggested SA activity based on engagement type:
     - WAFR → "Well Architected [Architecture]"
     - GameDay → "GameDay [Workshops]"
     - Landing Zone Assessment Accelerator → "Well Architected [Architecture]"
     - SIP, SHIP, SCVR → "Security, Resilience and Compliance [Architecture]"
     - ReSCO (Security/Resilience) → "Well Architected [Architecture]"
     - ReSCO (Cost Optimization) → "Cost Optimization [Management]"
     - Partner engagement → "Partner Solution Engagement [Architecture]"
     - ProServe engagement → "Support of Proserve Engagement [Management]"
     - Default → "Meeting / Office Hours [Management]"
   - Allow user to override
   - **For ReSCO:** Confirm compliance with ReSCO tracking guidelines before proceeding

4. **Services and Domains**
   - Prompt for AWS services involved (optional)
   - Prompt for technical domains (optional)
   - Suggest based on engagement type:
     - Security engagements → "Security, Identity, & Compliance" domain
     - Resilience engagements → "Resilience" domain
   - **Can be skipped:** User doesn't need to provide if not applicable

5. **Create Tech Activity**
   - **parentRecord**: opportunity ID or account ID
   - **activityDate**: completion date
   - **status**: "Completed"
   - **subject**:
     - ReSCO activities: "FY-25-ReSCO - {Engagement Type} - {Customer}"
     - Other activities: "{Engagement Type} - {Customer}"
   - **description**: User-provided description OR auto-generated if skipped
   - **saActivity**: Selected activity type
   - **goals**:
     - ReSCO Security/Resilience: ["G3 2025 - ReSCO - SSR"]
     - Other G3 engagements: ["G3 2025 - {selected engagement}"]
     - ReSCO Cost Optimization: NO G3 goal tag
   - **services**: User-provided services (optional)
   - **domains**: User-provided domains OR auto-suggest based on engagement type
   - **isVirtual**: true (default)

6. **Validation**
   - If opportunity-based, check:
     - ✅ Valid estimated value (not $0, $1, or placeholder)
     - ✅ Opportunity products listed
     - Warn if missing but allow proceed

7. **Confirmation**
   - Show Salesforce URL for created activity
   - Show G3 goal tag applied
   - Offer to track another engagement

## Description Guidelines

**User can type "skip" to auto-generate a basic description.**

**DO include:**
- Impact and value delivered
- Key outcomes and findings (high-level)
- Next steps and action items
- Customer feedback or testimonials
- Business benefits achieved
- **For ReSCO:** Pillar name, number of accounts scanned, number of critical/medium/low findings, recommendations

**DO NOT include:**
- Specific security vulnerabilities
- Detailed risk findings
- Sensitive customer data
- Compliance violation details
- Confidential security configurations

**Auto-generated description template (when user skips):**
- Landing Zone: "Conducted Control Tower multi-account setup across multiple regions for {Customer}. Established foundational landing zone with centralized governance and guardrails."
- SIP: "Conducted Security Improvement Program (SIP) assessment for {Customer}. Identified security enhancements and provided actionable recommendations for AWS infrastructure security posture improvement."
- Generic: "Conducted {Engagement Type} for {Customer}. Provided {Security/Resilience} guidance and recommendations."

## Implementation Notes

- Use aws-sentral MCP tools:
  - `search_opportunities` or `search_accounts` - find target
  - `get_opportunity_details` - verify opportunity info
  - `create_tech_activity` - create activity with goals
  - `get_opportunity_line_items` - verify products listed

- Goals parameter format:
  ```python
  # Standard G3 engagement
  goals = ["G3 2025 - AWS Well-Architected Framework Review (WAFR) - SSR"]

  # ReSCO Security/Resilience
  goals = ["G3 2025 - ReSCO - SSR"]

  # ReSCO Cost Optimization (NO goals parameter)
  # Do not include goals field for Cost Optimization
  ```

- Always set:
  - `status`: "Completed"
  - `isVirtual`: true (default)
  - `goals`: Array with one G3 2025 goal (except Cost Optimization)

- Subject line format:
  - ReSCO activities: MUST start with "FY-25-ReSCO - "
  - Other activities: "{Engagement Type} - {Customer}"

- Date validation:
  - Must be in 2025
  - Must be <= today
  - Default to today if not specified
  - Accept weekday hints (e.g., "early November, not a weekend")

- Domain auto-suggestion:
  - Security engagements → "Security, Identity, & Compliance"
  - Resilience engagements → "Resilience"
  - Control Tower/Landing Zone → "Security, Identity, & Compliance"

- AskUserQuestion for engagement selection:
  - Show categorized list (Security vs Resilience)
  - Allow "more" option to show full list
  - Provide clear numbering

- ReSCO validation:
  - When user mentions ReSCO or FY-25-ReSCO, confirm activity type mapping
  - Before creating activity, verify compliance with ReSCO guidelines
  - Ask: "does this comply with the resco tracking guidelines"

## Error Handling

- If opportunity/account not found: ask for different search term
- If invalid date: prompt for correct format
- If description too short: prompt for more detail
- If missing products on opp: warn but allow proceed
- If activity creation fails: show error and offer retry
