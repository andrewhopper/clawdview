## Instructions

1. **Locate style guide**:
   - Look for `STYLE_GUIDE.md`, `WRITING_STYLE_GUIDE.md`, or similar in repo root
   - If not found, use general technical writing best practices

2. **Read the file**: Use Read tool to load `{file_path}`

3. **Check against standards**:
   - Active voice usage (goal: >80%)
   - Citation presence and format

## What to Check

### Active Voice
Count passive constructions like "was announced", "is being", "has been". Should be <20% of sentences.

### Citations
- Factual claims (numbers, dates, names) should have [1], [2], etc.
- Check for "Data Sources" or "References" section
- Citations should map to sources

### Specificity
Flag vague words:
- "Recently", "soon", "many", "several"
- "Large", "small", "growing"
- "Leading", "innovative", "cutting-edge"
- Suggest concrete replacements

### Paragraph Length
Count sentences per paragraph. Flag if >6 sentences.

### Jargon/Buzzwords
Flag: "leverage", "synergy", "best-in-class", "world-class", "innovative", "cutting-edge", "disruptive"

### Formatting
- Proper heading hierarchy (H1 → H2 → H3)
- Bullet points formatted correctly
- No broken markdown syntax
- Consistent style

## Example Output

Show line numbers, current text, and specific suggested fixes. Be helpful, not harsh.

Present your report clearly formatted in markdown.
